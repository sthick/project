After we started with two new data collection devices, FlightScope and Rapsodo, there was an extreme amount of data that the coaches did not know how to use. This information overflow made it hard for them to parse through the data and look at numbers they wanted to analyze. The pitching coach and video coordinator approached me with this issue and requested an excel file that does easy calculations on a limited number of the data sets. In response I created a macro based excel file that allowed the pitching coach to look more closely at the data and analyze it based on pitcher, date, and more.

I am currently researching pitch spin for pitchers and exit velocity for hitters. These are two of the main measurements the coaches look at on a daily basis and with almost three years of data for me to look at I have decided to look at the change in these metrics throughout a season. I hope to find a relationship of when pitchers and hitters are at their best performance levels in order to adjust routines to make sure when they are at their best it is at the right time in the season.

Part 1:
I was given the job to pull data from LeagueApps camp data that was unusable by the marketing department because of the form the data was in. I did this by creating a lambda function to take relevant information gathered through camp/clinic registration from LeagueApps API and store it in the company's Amazon S3 bucket as a json file for future use. This process previously was done every morning by a marketing employee who had to log into the LeagueApps website, download the necessary information as a csv file, clean the data to make it readable removing unnecessary information and checking for duplicate registrationIDs, and saving it for future use. With the creation of this lambda there is no longer a need for manual labor because after deploying the code it will run every morning saving the information to the necessary location in an easily scrapable file as well as a file that is easy to read and access.

Part 2:
After pulling LeagueApps information we now wanted to add this information to the customer profile information the Cavaliers have been accumulating for years. I was assigned with adding the new information from LeagueApps to the customer profile as well as updating the customer profile information that was already there. The program I created pulls the customer profile information and LeagueApps information and checks for duplicates, updated existing information, and adds the the new contacts to the profile.

In preparation for a history of the Cleveland Cavaliers website a history of each team for the last fifty years was created and saved in a google spreadsheet. Each team had its own worksheet filled with information from that year (e.g. Players, coaches, uniforms, ect). This google spreadsheet needed to be added to an Amazon S3 bucket as a json file in order to be scraped and used for website creation. I was approached with the task of finding the easiest, and cheapest way to update an S3 bucket with a file for each year every time a change was made to that particular google spreadsheet. The program I wrote allows that to happen because once deployed the runtime was set to every 4 seconds, therefore if a change is made the S3 bucket will be updated with that change almost immediately.
